## S Tier - Game changers. Use these daily or you're leaving value on the table.

**Claude Code** - Reads your codebase, writes code, runs terminal commands, commits to git. Not a chatbot. A coding agent. Most new code at Anthropic is written through it. S

**Projects** - Set instructions once, every chat in that project follows them. Upload your docs, style guide, whatever. Claude stops asking "what do you do?" every conversation. S

**Deep Research** - Runs for minutes, pulls dozens of sources, returns a full research brief. Replaced Perplexity for me. What takes an hour of Googling takes one prompt. S

**MCP** - Connects Claude to your tools. Google Drive, Slack, GitHub, Notion, databases. 6,000+ integrations. Without it Claude is a chatbot. With it Claude is a workflow. S

![Image](https://pbs.twimg.com/media/HF2brqXbAAA1VXj?format=jpg&name=large)

## A Tier - Seriously good. Not daily use for everyone, but when you need it, nothing else comes close.

**Artifacts** - Claude generates files right in the chat. HTML pages, charts, SVGs, React components, documents. You see the result rendered live. A

**Web Search** - Searches the internet, cites sources. Not as deep as Deep Research but faster. Good for quick fact-checks and "what's the latest on X." A

**Memory** - Remembers your preferences across sessions. Free tier included. Still early, sometimes forgets obvious things, but when it works you stop repeating yourself. A

**Extended Thinking** - Shows reasoning step by step before answering. For hard problems (complex code, math, analysis) the quality jump is real. A

**Skills & Slash Commands** - Save a prompt as a markdown file, invoke it with /name. Write once, reuse forever. /review, /test, /commit instead of typing 50 words every time. A

![Image](https://pbs.twimg.com/media/HF2glZjWkAA6bHg?format=jpg&name=large)

## B Tier - Solid features. Useful for specific workflows, not essential for everyone.

**Hooks** - Auto-actions on events: format code on every edit, lint before commits, notify when done. Unlike CLAUDE.md (advisory), hooks fire 100% of the time. B

**Plugins** - One-click bundles of skills and tools. 400+ in the marketplace. Frontend Design alone has 400K installs. Pick 3-5 for your workflow, skip the rest. B

**Styles** - Save custom writing tones. "Technical blog" or "casual email." Useful for content work but most people just describe the tone in the prompt. B

**Agent Teams** - Multiple Claude agents working in parallel. Each in its own git worktree. Powerful but most tasks don't need parallelization. B

**Cowork** - Claude Code but for non-developers. Desktop app, manages files, controls browser. Early but impressive. B

**1M Context Window** - 750K+ words in one conversation. Most people never hit 200K. But when you need to process an entire codebase or contract at once, nothing else does this. B

![Image](https://pbs.twimg.com/media/HF2pXlUXoAA_VUK?format=jpg&name=large)

## C Tier - Nice to have. Works fine but you won't miss it if it's gone.

**Voice Mode** - Talk to Claude with push-to-talk. 20 languages. Cool but typing is still faster for most tasks. C

**Claude Apps** - Build small apps powered by Claude. Limited compared to Projects with custom instructions. Most people set one up and never open it again. C

**Batch API** - Thousands of requests at 50% discount. Great if you're building a product on Claude. Irrelevant for everyone else. C

**Prompt Caching** - 90% cost savings on repeated context. API-only. Most people never touch it directly. C

![Image](https://pbs.twimg.com/media/HF2gpN0XoAAb--b?format=jpg&name=large)

## D Tier - Exists. Technically works. Rarely worth the effort.

**Computer Use** - Claude sees your screen and clicks things. Sounds amazing. In practice, slow and breaks on anything non-trivial. Party trick, not a workflow. D

**Remote Control** - Control Claude Code from your phone. Great concept. But debugging code on a 6-inch screen is not it. D

**Model Switching Mid-Chat** - Switch between Haiku, Sonnet, Opus in one chat. Nobody actually does this in [Claude.ai](https://claude.ai/). D

## F Tier - Skip it. Seriously.

**Sharing Conversations** - Share a Claude conversation via link. The feature exists but almost nobody uses it. If you want to share Claude's output, you copy it. A link to a conversation is not how people consume information. F

**Claude for Slack (the old version)** - Before MCP, Claude had a basic Slack integration. It was slow, had limited context, and couldn't do much beyond answering questions in a channel. MCP's Slack integration replaced it entirely. F

## Where most people get stuck

The gap isn't between free and paid users. It's between people who treat Claude as a chatbot and people who treat it as infrastructure.

**If you're using Claude without Projects**, you're re-explaining yourself every conversation.

**If you're using Claude without MCP,** you're copy-pasting between tabs.

**If you're coding without Claude Code**, you're mass typing what a slash command does in one second.

Start with Projects. Set up one project with your context. Use it for a week, then add MCP, then try Deep Research.
