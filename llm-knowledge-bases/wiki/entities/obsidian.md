# Obsidian

## Summary

Obsidian is the primary human-facing frontend for this project. It is where the human browses the wiki, follows links, views structure, and inspects durable outputs, while the LLM maintains the underlying markdown files.

## Why relevant

Multiple sources converge on Obsidian as the right interface layer:
- it is a strong markdown frontend
- it supports graph view and plugins
- it is compatible with git-based file workflows
- it pairs naturally with an LLM-maintained knowledge base
